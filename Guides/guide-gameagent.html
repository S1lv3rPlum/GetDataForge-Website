This guide explains how the DataForge Game Agent works, what you are watching when you run it, and how to get the most out of the live dashboard. No programming experience required — everything is explained in plain language.

What is Reinforcement Learning?

Reinforcement learning is a way of teaching an AI by letting it try things and learn from the results — the same way a person learns to ride a bike or play a sport. The agent takes an action, the environment responds, and the agent receives a reward or penalty depending on how well it did.

Over thousands of repetitions, the agent builds up an understanding of which actions lead to good outcomes. It does not start with any knowledge of the game — it figures everything out from scratch through experience.

This is the same fundamental approach used by DeepMind to build AlphaGo and by OpenAI to build game-playing agents that beat professional human players.

The Two Agents

🎲 Random Agent

The Random Agent picks cells completely at random on every move. It has no memory, no strategy, and learns nothing. Its only purpose is to serve as a baseline — a measure of what pure luck looks like. On a standard Beginner board it wins very rarely, and its win rate stays flat no matter how many games it plays.

Running the Random Agent first is valuable because it shows you exactly what the Learning Agent has to beat. Any improvement you see in the Learning Agent above the Random Agent's baseline is genuine learned intelligence.

🧠 DQN Learning Agent

The DQN Learning Agent uses a neural network — a mathematical structure loosely inspired by the human brain — to decide which cell to click. At the start it behaves similarly to the Random Agent because its network has no knowledge. Over hundreds of games it gradually learns which moves tend to be safe and which tend to hit mines.

DQN stands for Deep Q-Network. The Q refers to a quality score the network assigns to each possible action. The agent picks the action with the highest quality score, and the network updates those scores after every game based on what actually happened.

How Minesweeper Works for the Agent

The agent sees the board as a grid of numbers. Hidden cells show as -1. Flagged cells show as -2. A revealed mine shows as -3. Revealed safe cells show the number of adjacent mines, exactly as a human player would see.

On each turn the agent chooses one of two types of action for any cell on the board — reveal it or flag it as a mine. This gives the agent a much larger decision space than a human clicking with a mouse, which is part of why learning takes many episodes.

Reward Structure

The agent learns through rewards and penalties applied after each action:

  • Reveal a safe cell: +1 point
  • Correctly flag a mine: +5 points
  • Incorrectly flag a safe cell: -3 points
  • Hit a mine: -10 points
  • Win the game: +20 points
  • Each step taken: -0.1 points (encourages efficiency)
  • Repeated or invalid action: -0.5 points

The agent does not know these rules explicitly — it discovers them through experience. After enough games it starts to understand that flagging a mine before revealing near it is a good strategy, because the reward signal reinforces that behavior.

Difficulty Levels

Three difficulty levels are available, matching standard Minesweeper conventions:

  • Beginner: 9x9 grid, 10 mines, 71 safe cells
  • Medium: 14x14 grid, 25 mines, 171 safe cells
  • Hard: 24x24 grid, 50 mines, 526 safe cells

The Learning Agent saves its progress separately for each difficulty level. Training on Beginner does not affect the Medium or Hard models. Starting on Beginner is strongly recommended — the agent learns faster on smaller boards and the results are more visible in the dashboard.

Reading the Live Dashboard

The dashboard opens automatically alongside the game window and updates after every episode. Here is what each panel shows:

Win Rate Over Time

The main graph at the top. Shows the percentage of games won as a running average. The Random Agent line stays low and flat. The Learning Agent line should gradually trend upward as training progresses — this is the most satisfying thing to watch. Both agents appear on the same graph in contrasting colors when running simultaneously.

Reward Per Episode

A bar chart showing total reward earned each game. Green bars are positive reward episodes. Red bars are losses. Early in training almost all bars are red. As the agent improves, green bars become more frequent and taller.

Steps Per Episode

How many moves the agent made before the game ended. A dotted yellow line shows the 10-episode rolling average. More steps generally means the agent survived longer — so watching this number increase over time is a sign of genuine improvement even before wins appear.

Correct Flags Per Episode

How many mines the agent correctly identified and flagged each game. A Learning Agent that is improving will flag more mines correctly over time as it starts to recognize safe and dangerous patterns.

Stats Panel

Bottom right. Shows a running summary for each agent — total episodes, wins, losses, win rate percentage, high score, and best winning streak.

Reading the Game Window

The game window shows the board updating in real time as the agent plays. Here is what the colors and symbols mean:

  • Gray cell: Hidden — not yet revealed
  • Dark cell with number: Revealed safe cell — number shows adjacent mine count
  • Yellow cell with 🚩: Flagged by the agent as a suspected mine
  • Red cell with 💣: Mine — game over

Two progress bars run along the bottom of the game window. The yellow bar tracks how many mines have been correctly flagged out of the total. The green bar tracks what percentage of safe cells have been revealed. A winning game fills the green bar completely.

Running Both Agents Simultaneously

Selecting Both from the agent selection screen launches the Random Agent and the Learning Agent at the same time, each in its own game window. Both agents write their performance data to a shared file, and both dashboards display data from both agents on the same graphs.

This is the most informative way to run the program. Watching the Learning Agent's win rate line gradually separate from the Random Agent's flat baseline is a direct visual demonstration of the AI learning in real time.

Tips for Getting the Most Out of It

  • Start with Beginner difficulty. The agent learns faster on smaller boards and improvements show up sooner in the dashboard.
  • Let it run for at least 500 episodes. The first 200-300 episodes look mostly random. Real improvement typically becomes visible around episode 300-500 as the neural network accumulates enough experience.
  • Watch the Steps graph, not just wins. The agent may go hundreds of episodes without a win but still be learning — surviving longer each time. Steps per episode trending upward is a meaningful signal of progress.
  • Run Both agents for the best comparison. Seeing the Learning Agent on the same graph as the Random Agent makes improvement much more visible than watching either one alone.
  • Progress saves automatically. Close the window at any time — the model saves every 100 episodes. The next run picks up where the last one left off.

Troubleshooting

The dashboard does not open

Make sure no other instance of the program is already running. Close all windows and relaunch from the launcher.

The Learning Agent does not seem to be improving

This is normal for the first several hundred episodes. Reinforcement learning requires a large number of games before patterns become consistent. Give it at least 500 episodes before drawing conclusions. Watch the Steps per episode graph — if that is trending upward the agent is learning even if wins are not showing yet.

The program runs slowly

Training speed depends on your hardware. A machine without a dedicated NVIDIA GPU will train on CPU only, which is slower. This does not affect the final result — it just takes longer to see improvement.
